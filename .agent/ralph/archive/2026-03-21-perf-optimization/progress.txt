## US-029: 再レンダリング抑制と TBT/INP 改善
**Date:** 2026-03-21

### Implementation
1. **DirectMessagePage 1ms setInterval → MutationObserver** (`DirectMessagePage.tsx`)
   - `setInterval(1)` + `window.getComputedStyle()` による毎ms のレイアウト強制再計算を廃止
   - `MutationObserver` で DOM 変更時のみ `scrollTo` を実行するよう変更
   - `scrollHeightRef` 不要になったため削除

2. **use_has_content_below 1ms scheduler polling → IntersectionObserver** (`use_has_content_below.ts`)
   - `scheduler.postTask` による 1ms 再帰ループ + `getBoundingClientRect()` 2回/ms を廃止
   - `IntersectionObserver` でコンテンツ末尾の可視状態をブラウザネイティブに監視
   - CrokPage のスクロールボタン表示に使用 — 不要な再レンダリングを排除

3. **use_search_params 1ms scheduler polling → React Router useSearchParams** (`use_search_params.ts`)
   - `scheduler.postTask` による 1ms ポーリング + `URLSearchParams` 生成を廃止
   - React Router 7 の `useSearchParams` をラップし、ルーター管理のイベント駆動に変更
   - SearchContainer での不要な再レンダリングを排除

4. **ChatInput サジェスト API デバウンス追加** (`ChatInput.tsx`)
   - キー入力ごとの即時 API コール (`/api/v1/crok/suggestions`) を 300ms デバウンスに変更
   - `setTimeout` + cleanup で前リクエストをキャンセル

5. **SoundPlayer onTimeUpdate を ref ベースに変更** (`SoundPlayer.tsx`)
   - `useState(currentTimeRatio)` → `useRef(overlayRef)` で DOM 直接更新に変更
   - ~4回/秒の `setCurrentTimeRatio` による再レンダリングを完全排除
   - オーバーレイ div の `style.left` を ref 経由で直接操作
