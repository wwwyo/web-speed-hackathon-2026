#13 GIF → MP4 変換

影響指標: LCP, 転送量 | 難易度: 中級

1. GIF が重い構造的な理由

GIF の仕組み: ┌──────────────────────────────────────────┐ │ フレーム1 │ フレーム2 │ フレーム3 │ ... │ │ (完全な画像)│ (完全な画像)│ (完全な画像)│ │ └──────────────────────────────────────────┘ 圧縮: LZW（フレーム間圧縮なし、256色制限） MP4 (H.264) の仕組み: ┌──────────────────────────────────────────┐ │ I-frame │ P-frame │ P-frame │ I-frame │ ... │ │ (完全) │ (差分) │ (差分) │ (完全) │ │ └──────────────────────────────────────────┘ 圧縮: フレーム間予測 + DCT + エントロピー符号化
GIFMP4 (H.264)
フレーム間圧縮なし（各フレームが独立）あり（差分のみ保存）
色数256色1677万色
圧縮率低い非常に高い
実際のサイズ10-26MB/ファイル60-325KB/ファイル

WSH 2026 では15個のGIF（合計179MB）を MP4（合計2.4MB）に変換し、98.7%のサイズ削減を達成しました。

2. ffmpeg での変換コマンド

# GIF → MP4 変換（Web最適化）
ffmpeg -i input.gif \
  -movflags +faststart \  # moov atom を先頭に移動（ストリーミング再生対応）
  -pix_fmt yuv420p \      # 互換性の高いピクセルフォーマット
  -vf "scale=trunc(iw/2)*2:trunc(ih/2)*2" \  # 偶数サイズに調整（H.264要件）
  output.mp4

重要なオプション解説

オプション目的なぜ必要か
-movflags +faststartmoov atom（メタデータ）をファイル先頭に配置デフォルトではファイル末尾にあり、全ダウンロード完了まで再生開始できない
-pix_fmt yuv420pピクセルフォーマット指定デフォルトの yuv444p は一部ブラウザで再生不可
-vf "scale=..."解像度を偶数に揃えるH.264 は奇数サイズの動画をエンコードできない

3. HTML側の実装変更

Before: gifler ライブラリで canvas に描画
import { Animator, Decoder } from "gifler";
import { GifReader } from "omggif";

// バイナリを fetch → GIF デコード → canvas に描画
const data = await fetchBinary(src);
const reader = new GifReader(new Uint8Array(data));
const frames = Decoder.decodeFramesSync(reader);
const animator = new Animator(reader, frames);
animator.animateInCanvas(canvasElement);
After: ネイティブ video タグ
<video
  ref={videoRef}
  src={src}
  autoPlay       // 自動再生
  loop           // ループ再生
  muted          // ミュート（autoPlay に必須）
  playsInline    // iOS で全画面再生しない
/>

4. autoPlay + muted が必須な理由

ブラウザの自動再生ポリシーにより、音声付きの動画は自動再生できません。muted 属性を付けることで、ユーザー操作なしでの自動再生が許可されます。GIF の代替なので音声は不要であり、muted は自然な選択です。

5. prefers-reduced-motion 対応

// アクセシビリティ: 視覚効果を減らす設定のユーザーには自動再生しない
useEffect(() => {
  const video = videoRef.current;
  if (!video) return;
  if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) {
    video.pause();
    setIsPlaying(false);
  }
}, []);

6. 削除できた依存

  • gifler — GIF アニメーションライブラリ
  • omggif — GIF デコーダー
  • gifler.d.ts — 型定義ファイル
  • fetchBinary の呼び出し — バイナリデータの取得が不要に

video タグはブラウザネイティブなので、追加ライブラリなしでハードウェアアクセラレーションによる高効率デコードが可能です。

確認クイズ: -movflags +faststart を忘れるとどうなる？

moov atom（動画のメタデータ: 解像度、コーデック情報、フレームのオフセット）がファイル末尾に配置されます。ブラウザはまずこのメタデータを読まないと再生を開始できないため、ファイル全体をダウンロードするまで再生が始まらないことになります。faststart を付けるとメタデータが先頭に移動し、ダウンロード開始直後から再生できます。

確認クイズ: GIF ではなく WebM や AVIF アニメーションではダメ？

WebM (VP9) は MP4 よりさらに小さくなる場合がありますが、Safari の古いバージョンでサポートが不完全です。AVIF アニメーションは最も圧縮率が高いですが、ブラウザ対応が限定的です。WSH では Chrome 最新版だけ動けばよいので WebM や AVIF も選択肢ですが、MP4 (H.264) が最も安全で変換も簡単です。
