#30 重いライブラリの軽量化

影響指標: Bundle Size | 難易度: 中級

1. Tree Shaking が効かないライブラリ

Tree Shaking は「使っていないコードをバンドルから除外する」最適化ですが、一部のライブラリは構造的に Tree Shaking が効きません。

Tree Shaking が効くライブラリ: // ESM + 個別 export import { Button } from "ui-lib"; // → Button のコードだけバンドルに含まれる Tree Shaking が効かないライブラリ: // デフォルト export が全言語定義を含むオブジェクト import SyntaxHighlighter from "react-syntax-highlighter"; // → 200以上の言語定義が全てバンドルに含まれる (1.2MB)

2. react-syntax-highlighter の問題

デフォルト import すると全ての言語定義がバンドルに含まれます:

Before: 1.262 MiB
import SyntaxHighlighter from "react-syntax-highlighter";
import { atomOneLight } from "react-syntax-highlighter/dist/esm/styles/hljs";

// ← これだけで 200+ 言語の文法定義が全て含まれる

3. prism-light パターン

react-syntax-highlighter には軽量版のエントリポイント prism-light があり、必要な言語だけを手動で登録できます。

After: 482 KiB (62%削減)
// 軽量版のベースコンポーネント
import SyntaxHighlighter from "react-syntax-highlighter/dist/esm/prism-light";

// 必要な言語だけ個別 import
import javascript from "react-syntax-highlighter/dist/esm/languages/prism/javascript";
import typescript from "react-syntax-highlighter/dist/esm/languages/prism/typescript";
import python from "react-syntax-highlighter/dist/esm/languages/prism/python";
import json from "react-syntax-highlighter/dist/esm/languages/prism/json";
// ... 必要な言語だけ

// スタイルも prism 用を使う
import { oneLight } from "react-syntax-highlighter/dist/esm/styles/prism";

// 手動で言語を登録
SyntaxHighlighter.registerLanguage("javascript", javascript);
SyntaxHighlighter.registerLanguage("typescript", typescript);
SyntaxHighlighter.registerLanguage("python", python);
SyntaxHighlighter.registerLanguage("json", json);

// エイリアス設定（"js" → "javascript" など）
SyntaxHighlighter.alias("javascript", ["js"]);
SyntaxHighlighter.alias("typescript", ["ts"]);

4. 未対応言語のフォールバック

// サポート言語のセットを作る
const supportedLanguages = new Set([
  "javascript", "typescript", "python", "json",
  "js", "ts", "py",  // エイリアスも含める
]);

// 使用時
if (!supportedLanguages.has(language)) {
  // 未対応言語は素の pre/code で表示（ハイライトなし、でも表示は崩れない）
  return <pre><code>{code}</code></pre>;
}

return (
  <SyntaxHighlighter language={language} style={oneLight}>
    {code}
  </SyntaxHighlighter>
);

6. バンドルサイズの確認方法

# Rsdoctor でビルド分析（WSH 2026 で導入済み）

# webpack-bundle-analyzer でも可
npx webpack-bundle-analyzer dist/stats.json

# 特定チャンクに含まれるライブラリを grep で確認
rg "react-syntax-highlighter" dist/
確認クイズ: hljs と prism の違いは？

react-syntax-highlighter は内部で2つのハイライトエンジンをサポートしています:

  • hljs (highlight.js): dist/esm/styles/hljs/ のスタイルを使う
  • prism (PrismJS): dist/esm/styles/prism/ のスタイルを使う。軽量版（prism-light）がある

Before は hljs、After は prism-light。エンジンの変更なのでスタイル名も変える必要があります（atomOneLight → oneLight）。
