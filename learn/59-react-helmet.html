#59 react-helmet の脱却

影響指標: Bundle Size | 難易度: 基礎

1. react-helmet とは何だったか

react-helmet は React コンポーネントから <head> 内の <title>、<meta>、<link> 等を宣言的に管理するライブラリです。

// react-helmet の使い方
import { Helmet } from "react-helmet";

function MyPage() {
  return (
    <>
      <Helmet>
        <title>ページタイトル</title>
        <meta name="description" content="説明" />
      </Helmet>
      <div>ページの中身</div>
    </>
  );
}

2. React 19 でネイティブサポートされた

React 19 から、JSXの中で直接 <title>、<meta>、<link> を返すと、React が自動的に <head> に配置してくれます（hoisting）。

React 19 の Document Metadata: function MyPage() { return ( <> <title>ページタイトル</title> ← React が自動的に <head> へ移動 <meta name="description" .../> ← 同様 <div>ページの中身</div> </> ); } レンダリング結果: <html> <head> <title>ページタイトル</title> ← ここに配置される <meta name="description" .../> </head> <body> <div>ページの中身</div> </body> </html>

3. WSH 2026 での移行

Before: react-helmet
// 各コンテナで Helmet を使用
import { Helmet } from "react-helmet";

const TimelineContainer = () => {
  return (
    <>
      <Helmet><title>CaX</title></Helmet>
      <TimelinePage ... />
    </>
  );
};

// AppContainer で HelmetProvider をラップ
import { HelmetProvider } from "react-helmet-async";
<HelmetProvider>
  <App />
</HelmetProvider>
After: PageTitle コンポーネント
// 薄いラッパー: <title> を返しつつ document.title も同期
const PageTitle = ({ title }: { title: string }) => {
  useEffect(() => {
    document.title = title;  // SPA遷移時にも確実に更新
  }, [title]);
  return <title>{title}</title>;
};

// 各コンテナで使用
const TimelineContainer = () => {
  return (
    <>
      <PageTitle title="CaX" />
      <TimelinePage ... />
    </>
  );
};

// HelmetProvider は不要！

4. なぜ document.title も設定するのか

React 19 の <title> hoisting は初回レンダリングでは動きますが、SPA でクライアントサイドルーティングする場合、非同期 fetch 後にタイトルが確定するケースがあります。

SPAでの落とし穴
const PostContainer = () => {
  const { data: post } = useFetch(`/api/v1/posts/${id}`);

  // data が null の間、title が空になる
  return (
    <>
      <title>{post?.text ?? ""}</title>   {/* ← fetch完了前は空 */}
      <PostPage post={post} />
    </>
  );
};

document.title を useEffect で同期すれば、fetch 完了後に確実にタイトルが更新されます。

5. 削除できたもの

パッケージサイズ
react-helmet~10KB (minified)
react-helmet-async~5KB
HelmetProvider ラッパーコンテキスト生成のオーバーヘッド

代わりに追加したのは PageTitle コンポーネント（15行）だけ。外部依存ゼロです。

6. React 19 で直接使える head 要素

要素React 19 で hoisting備考
<title>対応最後にレンダーされたものが勝つ
<meta>対応name/property で重複排除
<link>対応rel="stylesheet" はレンダーブロッキング扱い
<style>対応precedence で順序制御
<script>対応async は非ブロッキング
確認クイズ: react-helmet-async と react-helmet の違いは？

react-helmet は内部で副作用（直接 document.head を操作）を持ち、SSR で問題がありました。react-helmet-async はこれを修正し、React 18 の Concurrent Mode でも安全に動作するように Provider パターンを導入したフォークです。React 19 のネイティブ対応により、どちらも不要になりました。
