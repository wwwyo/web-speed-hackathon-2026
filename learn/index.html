SecurityTBT

860 → 950+ への戦略

  • SSR/Streaming HTML の導入で FCP/LCP をさらに短縮（最大+40点）
  • AVIF 画像フォーマットの採用で転送量を WebP からさらに30%削減
  • Speculation Rules API による次ページの先読み
  • INP の徹底的な改善（Interaction to Next Paint、200ms以内を目指す）
  • Critical CSS のインライン化で render-blocking を完全除去
  • Service Worker による戦略的プリキャッシュ（ただしSW内の処理を重くしない）
